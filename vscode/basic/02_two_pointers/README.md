# Two Pointers

> 2 つのインデックスを同時に動かして、ネストループ O(n^2) を O(n) に落とすテクニック。

---

## 2 つのパターン

### 1. 同方向 (slow / fast)

```
index :   0   1   2   3   4   5   6
array : [ a   b   c   d   e   f   g ]
          s→  f→→
              s→      f→→
                  s→          f→→
                      s→              f→→  (fast が末尾に着いたとき slow は真ん中)
```

代表用途:
- 連結リストの中央ノードを探す (`middleNode`)
- サイクル検出 (Floyd の Tortoise & Hare)
- リスト先頭からの「k 個前」を保つ (delete N-th from end など)

### 2. 反対方向 (left / right)

```
index :   0   1   2   3   4   5   6
array : [ a   b   c   d   e   f   g ]
          L→                      ←R
              L→              ←R
                  L→      ←R
                      LR              (L >= R で停止)
```

代表用途:
- 回文判定 (`isPalindrome`)
- ソート済み配列で「和が target になるペア」(Two Sum II)
- Container With Most Water / Trapping Rain Water

---

## 計算量サマリ

| 種類          | 時間  | 空間  |
| ------------- | ----- | ----- |
| 同方向 slow/fast | O(n)  | O(1)  |
| 反対方向 L/R   | O(n)  | O(1)  |

ネストループの素朴解 O(n^2) → ポインタ 2 本で **1 パス O(n)** にできるのが旨味。

---

## いつ気づくか (見分けポイント)

1. 入力が **ソート済み配列 / 文字列 / 連結リスト** で、和・差・対称性を問われている → 反対方向。
2. 「中央」「サイクル」「末尾から k 番目」など **「片方が先に進む」必要** がある → 同方向 (slow/fast)。
3. ネストループで `O(n^2)` を書きそうになり、かつ「内側ループが外側に依存して単調に動く」と気づいた → 2 ポインタに置き換え可能。
4. ハッシュマップで解けるが O(n) 空間を使うのがもったいない時 → 並べ替えてから 2 ポインタ、で空間 O(1) に落ちる。
