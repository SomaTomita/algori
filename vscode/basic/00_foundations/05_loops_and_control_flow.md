# ループと制御フロー

## ほぼすべての問題はこの 3 要素

```
   ┌──────────┐
   │  ループ   │   配列・文字列・range を走査
   └────┬─────┘
        │
   ┌────▼─────┐
   │  条件     │   if / while で分岐
   └────┬─────┘
        │
   ┌────▼─────┐
   │ 変数更新 │   答え / ポインタ / カウンタを更新
   └──────────┘
```

> 大半のインタビュー問題 = **ループ + 条件 + 変数更新** の組み合わせ。

## range / enumerate / zip の使い分け

| 書き方                              | 用途                       | 例                              |
| ----------------------------------- | -------------------------- | ------------------------------- |
| `for x in arr`                      | 値だけ欲しい               | 合計、最大値                    |
| `for i in range(len(arr))`          | index だけ欲しい           | 2 つ先の要素を見る              |
| `for i, x in enumerate(arr)`        | index と値の両方           | 「何番目の x か」を返す         |
| `for a, b in zip(arr1, arr2)`       | 2 つの配列を並列に走査     | 要素同士の比較・ペア処理        |
| `while l < r:`                      | 条件で止まる / 二分探索    | two-pointer, binary search      |

```python
for i, x in enumerate(arr):       # 添字付きの走査が一番出番が多い
    if x == target:
        return i
```

## よくあるバグ

- ループポインタの **更新忘れ** (`while` で `i += 1` を書き忘れて無限ループ)
- 境界条件の off-by-one (`range(n)` vs `range(n+1)`)
- ループ内で扱う **配列を同時に変更** している

## 紙シミュレーションが最強

> 書いたコードは **小さな入力で手で 1 ステップずつ実行** する。
> ポインタが動く位置、変数が変わる瞬間を紙に書くとバグが消える。
