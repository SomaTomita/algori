# Recursion（再帰）

## 一言でいうと

**「自分自身を呼び出す関数」**。問題を「同じ形のより小さな問題」に分割するアプローチ。

## なぜ重要か

- 木 (Tree) と グラフ (Graph) の探索は、ループより再帰のほうが圧倒的に書きやすい。
- DFS、バックトラッキング、分割統治、DP のメモ化 — すべて再帰の応用。
- 「再帰が苦手」な人が「木は難しい」「バックトラックは難しい」と感じる。実は土台の再帰が固まっていないだけ。

## 再帰の 3 要素

```
def recurse(args):
    if 基底ケース:           # (1) いつ止まるか
        return 自明な答え
    
    部分問題で再帰呼び出し     # (2) 自分を小さくして呼ぶ
    
    return 結果を組み立てる   # (3) 部分問題の結果を統合する
```

| 要素 | 説明 | これが無いと |
|------|------|-------------|
| 基底ケース (Base case) | 再帰を止める条件 | 無限再帰 -> StackOverflow |
| 再帰ステップ | 自分自身を呼ぶ。引数は必ず「より小さく」 | 進まずに無限ループ |
| 統合 | 子の結果を親で組み立てる | 答えが返らない |

## 例: 階乗

```python
def factorial(n: int) -> int:
    if n <= 1:           # 基底ケース
        return 1
    return n * factorial(n - 1)   # 再帰ステップ + 統合
```

呼び出しの流れ:

```
factorial(4)
  = 4 * factorial(3)
  = 4 * (3 * factorial(2))
  = 4 * (3 * (2 * factorial(1)))
  = 4 * (3 * (2 * 1))
  = 24
```

## コールスタックの可視化

再帰呼び出しは **無料ではない**。各呼び出しごとに

- 引数
- ローカル変数
- 戻り先のアドレス

をスタックに積む。深さ N の再帰なら **O(N) の追加メモリ** を消費する。

```
factorial(4) を呼んだ時点のスタック:

    [factorial(1)]   <- 一番上 (top)
    [factorial(2)]
    [factorial(3)]
    [factorial(4)]   <- 一番下 (bottom)
```

Python は **デフォルトで再帰深さ 1000 まで**。それを超えると `RecursionError`。

```python
import sys
sys.setrecursionlimit(10**6)  # 必要なら拡張可能（推奨はしない）
```

## 計算量の考え方

再帰の計算量は **「呼び出し回数 × 1 回あたりの仕事量」** で見る。

| 例 | 呼び出し回数 | 1 回の仕事 | 合計 |
|----|-------------|-----------|------|
| 階乗 factorial(N) | N | O(1) | O(N) |
| ナイーブ Fibonacci | O(2^N) | O(1) | O(2^N) |
| メモ化 Fibonacci | N | O(1) | O(N) |
| 二分探索（再帰版） | log N | O(1) | O(log N) |

## 末尾再帰について

Python は **末尾再帰最適化をしない**。つまり

```python
def f(n, acc=1):
    if n == 0:
        return acc
    return f(n - 1, acc * n)   # 末尾呼び出しだが最適化されない
```

と書いても、スタックは普通に積まれる。深い再帰になる場合は反復に書き直す。

## 再帰 vs 反復

| 観点 | 再帰 | 反復 (loop) |
|------|------|-------------|
| 可読性 | 木・グラフでは圧勝 | 線形構造では同等 |
| メモリ | O(深さ) のスタック消費 | O(1) も可能 |
| 速度 | 関数呼び出しのオーバーヘッドあり | 高速 |
| 実装難易度 | 思考と実装が一致 | 状態を手動管理 |

**ルール**: 木・グラフ・分割統治 -> 再帰。配列の単純走査 -> 反復。

## アンチパターン

### NG 1: 基底ケースが間違っている

```python
def f(n):
    if n == 0:        # n が負になる経路があると無限再帰
        return 0
    return f(n - 2)
```

`f(3)` を呼ぶと 3 -> 1 -> -1 -> -3 ... と止まらない。
**「基底ケースは到達できるか」を必ず確認**。

### NG 2: 引数が小さくならない

```python
def f(n):
    if n == 0:
        return 0
    return f(n)   # 同じ引数で呼んでいる -> 無限ループ
```

### NG 3: 同じ部分問題を何度も計算

ナイーブ Fibonacci は `fib(n-1)` と `fib(n-2)` の中で同じ値を何度も再計算する。
**メモ化** (`@lru_cache` or 辞書で結果を保存) で O(N) に落とせる。

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n: int) -> int:
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)
```
